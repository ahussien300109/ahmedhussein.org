Introduction to Networks

A network connects devices to share data and resources. Understanding types, components, and reliability characteristics is the foundation of the CCNA exam.

Network Types Topology Components Reliability
Network Definition

A network is a group of two or more computers or electronic devices that are interconnected for the purpose of exchanging data and sharing resources. Every device on a network is called a host or end device.

Key Concept
The purpose of an IP address is to handle communication between devices that send and receive information across a network. The IP address uniquely identifies every device on the network.
Network Types
🏢
LAN — Local Area Network
Connects devices within a building or group of nearby buildings. Typically fast (1Gbps+). Managed by a single enterprise.
Enterprise Managed
🏫
CAN — Campus Area Network
Larger than a LAN, spans multiple nearby buildings. Typical in universities. Users share resources across buildings.
Multi-Building
🏙️
MAN — Metropolitan Area Network
Larger than LAN but smaller than WAN. Covers a city. Ownership and maintenance handled by an ISP.
ISP Managed
🌍
WAN — Wide Area Network
Connects networks across long physical distances. The Internet is the largest WAN. Managed by multiple administrators or the public.
Global Scale
Network Components
💻
End Devices (Hosts)
Servers provide services: email, web, file storage.
Clients send requests to servers to retrieve information.
🔄
Intermediary Devices
Switches, routers, firewalls, wireless APs. Regenerate signals, maintain path info, notify errors.
🔌
Network Media
Copper: Electrical impulses
Fiber-optic: Pulses of light
Wireless: Electromagnetic waves
DeviceOSI LayerFunction
SwitchLayer 2Forwards frames within a LAN using MAC addresses
RouterLayer 3Routes packets between networks using IP addresses
FirewallL3–L7Filters traffic based on security policies
Wireless APLayer 2Connects wireless hosts to a wired network segment
Reliable Network Characteristics
1
Fault Tolerance
Limits the impact of a failure by limiting the number of affected devices. Multiple paths are required for fault tolerance — if one path fails, traffic automatically reroutes.
2
Scalability
A scalable network can expand quickly and easily to support new users and applications without impacting performance of existing services.
3
Quality of Service (QoS)
The primary mechanism for reliable delivery of content. With a QoS policy, the router manages flow of data and voice traffic, prioritizing critical applications.
4
Network Security
Physical security: protecting network devices from unauthorized access. Information security: protecting data transmitted over the network.
Exam Mnemonic — FSQS
Remember the 4 reliable network characteristics: Fault Tolerance · Scalability · QoS · Security. These appear regularly in CCNA scenario questions.

Cisco IOS

Cisco IOS (Internetwork Operating System) is the CLI-based OS on all Cisco routers and switches. Mastering its modes, commands, and access methods is critical for the CCNA exam.

CLI Modes Access Methods Configuration Security
What Is Cisco IOS?

IOS is a CLI-based network operating system that enables a network technician to: use a keyboard to run CLI-based programs, enter text commands, and view output on a monitor.

⌨️
Console Port
Physical management port for local access. Used for initial configuration and when no network access is available.
Out-of-Band
🔒
SSH (Secure Shell)
Encrypted remote CLI connection via virtual interface. Recommended for all remote management.
Recommended
⚠️
Telnet
Insecure remote CLI connection. User authentication and commands are sent in plaintext over the network.
Insecure

IPv4 Addressing & Subnetting

Master IP address classes, binary conversion, and subnetting Class A, B, and C networks including VLSM. Subnetting is one of the highest-weighted topics on the CCNA exam.

Class A/B/C Binary Math Subnetting VLSM
IPv4 Address Classes

IPv4 addresses are 32-bit numbers written as 4 octets. The first octet value determines the class. Each class has a default subnet mask and private range.

Class1st OctetDefault MaskCIDRPrivate Range
A1 – 126255.0.0.0/810.0.0.0 – 10.255.255.255
B128 – 191255.255.0.0/16172.16.0.0 – 172.31.255.255
C192 – 223255.255.255.0/24192.168.0.0 – 192.168.255.255
D224 – 239N/AN/AMulticast
E240 – 255N/AN/AExperimental
Special Addresses
127.0.0.1 — Loopback (localhost)  |  0.0.0.0/0 — Unspecified / default route  |  169.254.x.x — APIPA (no DHCP server found)
Binary Conversion

Each octet = 8 bits. Bit values left to right: 128 · 64 · 32 · 16 · 8 · 4 · 2 · 1

Converting 168 → Binary (Step by Step)
Is 168 ≥ 128? YES → write 1, subtract: 168−128=40
Is 40 ≥ 64? NO → write 0
Is 40 ≥ 32? YES → write 1, subtract: 40−32=8
Is 8 ≥ 16? NO → write 0
Is 8 ≥ 8? YES → write 1, subtract: 0
Remaining bits = 0    Result: 168 = 10101000
Subnetting Formulas
2h ≥ Required Hosts
"h" = zero bits in new mask · Valid hosts = 2ʰ − 2
2n ≥ Required Subnets
"n" = one bits borrowed · Subnets = 2ⁿ
Block Size
The block size = value of the last borrowed bit (the last "1" in the subnet mask). It's the increment between subnet addresses. Example: /27 → last 1 is in the 32 position → block = 32.

VLSM — Variable Length Subnet Masking

VLSM allows using different subnet masks within the same network. This maximizes IP address efficiency by allocating exactly the right number of addresses to each subnet.

VLSM Strategy
Always subnet the largest requirement first, then work down to smaller subnets. This prevents overlapping and ensures efficient allocation.
VLSM Example — 192.168.1.0/24
Requirements: Dept A=50 hosts · Dept B=25 hosts · Dept C=10 hosts · WAN Link=2 hosts
# Step 1 — Largest first: Dept A (50 hosts) → 2^6=64 → /26 → block 64 192.168.1.0/26 → Hosts: .1–.62 · BID: .63 # Step 2 — Dept B (25 hosts) → 2^5=32 → /27 → block 32 192.168.1.64/27 → Hosts: .65–.94 · BID: .95 # Step 3 — Dept C (10 hosts) → 2^4=16 → /28 → block 16 192.168.1.96/28 → Hosts: .97–.110 · BID: .111 # Step 4 — WAN Link (2 hosts) → 2^2=4 → /30 → block 4 192.168.1.112/30 → Hosts: .113–.114 · BID: .115
